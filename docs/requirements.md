**版本**: v1.0  
**日期**: 2026-05-22  
**状态**: 已确认

---

## 一、产品目标

在信息量巨大的今天，快速找到有价值的资讯非常关键。本平台自动从多个渠道收集并分析资讯，帮助用户轻松掌握最新趋势和热点，提高工作效率和决策质量。

---

## 二、用户角色

| 角色 | 描述 |
|------|------|
| 普通用户 | 设置关键词、查看热点、接收通知 |
| 系统 | 定时采集、AI 分析、推送通知 |

---

## 三、核心功能

### 3.1 关键词监控

- 用户可手动添加/删除/启停关键词
- 每个关键词支持设置：
  - 名称（如 "Claude 4"）
  - 监控范围（如 "AI编程"、"科技新闻"）
  - 通知开关（浏览器 / 邮件）
  - 优先级（高/中/低）
- 当关键词相关内容出现时，**AI 先验证真实性**（过滤广告/营销/虚假内容），通过后第一时间推送通知
- 支持关键词分组管理

### 3.2 热点自动发现

- 用户设置关注领域（如 "AI编程"、"创业"、"科技"）
- 系统每 **30 分钟**自动采集一次
- 采集来源（8个并行，多维度覆盖，避免单一来源）：
  1. **Twitter/X** — twitterapi.io REST API，国外热点首发地（需 API Key）
  2. **Bing** — Bing News RSS 源，中英文兼顾，覆盖面广（无需 Key）
  3. **Google** — Google News RSS 源，全球最大搜索引擎（无需 Key）
  4. **DuckDuckGo** — HTML 搜索结果爬取，隐私友好（无需 Key）
  5. **Hacker News** — 官方免费 Firebase API，技术深度高（无需 Key）
  6. **搜狗微信** — 搜狗微信文章搜索爬取，独家微信公众号内容（无需 Key）
  7. **B站** — Bilibili 公开 JSON API，国内科技类视频丰富（无需 Key）
  8. **微博** — 微博热搜 API + 关键词搜索爬取，国内热点传播主阵地（无需 Key）
- AI 对每条内容进行：
  - 真实性评估（0-100分）
  - 关键词相关性评估（0-100分）
  - 热度评分（综合互动数据）
  - 摘要生成（50字以内）
- 低质量内容（评分 < 60）自动过滤不展示

### 3.3 热点 Feed 展示

- 瀑布流卡片式布局
- 按来源 Tab 切换：全部 / Twitter / Bing / Google / DuckDuckGo / HN / 搜狗 / B站 / 微博
- 每张卡片展示：
  - 标题 + 摘要
  - 来源标签（带图标）
  - AI 评分环形进度条
  - 发布时间（相对时间）
  - 互动数据（点赞、转推、评论）
  - 原文链接
- 支持手动刷新 + SSE 实时自动更新

### 3.4 通知系统

**浏览器推送（SSE）**：
- 实时连接，后端检测到命中关键词立即推送
- 浏览器 Tab 标题角标提示
- 页面内 Toast 通知（带来源和评分）

**邮件推送**：
- 用户在设置中填写邮箱地址
- 每次命中关键词发送一封 HTML 格式邮件
- 邮件包含：标题、摘要、来源、评分、原文链接
- 支持每日摘要模式（每天发一封汇总）

### 3.5 设置中心

- API Keys 配置（OpenRouter、twitterapi.io、Firecrawl）
- 邮件 SMTP 配置
- 通知频率控制（防止频繁打扰）
- 采集频率调整（默认30分钟）
- 数据保留策略（默认保留7天）

## 四、产品形式

### 4.1 Web 应用（第一阶段）

- **响应式设计**：兼容桌面、平板、手机
- **暗黑数据终端风格**：霓虹渐变 + 扫描线动效，与其他产品形成差异化
- **技术栈**：Next.js 16 全栈

### 4.2 Agent Skills（第二阶段，Web 完成后）

- 封装为可被其他 AI 调用的技能
- 支持：查询热点、添加关键词、获取分析结果
- 格式：符合 Claude Code Skill 规范

---

## 五、非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 页面首屏加载 < 2秒 |
| 可靠性 | 单个采集源失败不影响整体 |
| 安全性 | API Keys 仅存服务端，不暴露到前端 |
| 可维护性 | 采集引擎模块化，方便增减信息源 |
| 频率控制 | 爬虫采集每源间隔 ≥ 10秒，防止被封 |
| 数据保留 | 默认保留 7 天历史数据，自动清理 |

## 六、需要用户提供的配置

| 配置项 | 用途 | 获取方式 |
|--------|------|---------|
| `OPENROUTER_API_KEY` | AI 分析（真实性/热度评分/摘要） | openrouter.ai |
| `TWITTER_API_KEY` | Twitter/X 数据采集 | twitterapi.io（免费注册） |
| `FIRECRAWL_API_KEY` | 网页爬虫采集 | firecrawl.dev |
| `SMTP_HOST/USER/PASS` | 邮件推送 | Gmail / 163 / 企业邮箱 |
